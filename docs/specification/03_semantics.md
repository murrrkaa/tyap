# Семантика языка программирования

## 1. Области видимости
- Язык поддерживает **глобальную и локальную область видимости**:
    - Глобальная область видимости создаётся на уровне программы.
    - Локальная область видимости создаётся внутри функции.
- Локальные переменные и параметры доступны **только внутри функции**, после завершения функции они исчезают.
- Глобальные переменные доступны **во всей программе**, кроме случаев, когда их скрывают локальные переменные с тем же именем.

## 2. Уникальность имён
- Имена переменных, констант и функций должны быть уникальны **в пределах одной области видимости**.
- Локальная переменная может скрыть переменную внешней области видимости с тем же именем.
- Переменные и функции используют **общее пространство имён** внутри одной области видимости.
- Имена идентификаторов чувствительны к регистру.

## 3. Параметры функций
- Параметры передаются **по значению**.
- Локальная переменная не может скрыть имя параметра.
- Параметру **можно присвоить новое значение** внутри функции; это не влияет на аргумент вызова.
- Несовпадение числа формальных и фактических параметров недопустимо: число аргументов должно совпадать с количеством параметров функции.
- Выражения аргументов вычисляются **слева направо**.
- Функция не может завершаться без `return`, `return;` указывает на тип возвращаемого значение `void`

## 4. Рекурсия
- Поддерживается **прямая рекурсия**: функция может вызывать сама себя.
- Поддерживается **взаимная рекурсия**: две или более функций могут вызывать друг друга.

## 5. Ветвления
- Условие в `if` должно быть типа `bool`.
- Значение `true` → выполняется блок `if`, значение `false` → блок `else` (если он есть).

## 6. Циклы
- `while` выполняется, пока условие (тип `bool`) истинно.
- `for` имеет форму: `for (инициализация; условие; шаг)`
    - Инициализация выполняется один раз перед первым проходом.
    - Условие проверяется перед каждой итерацией.
    - Шаг выполняется после каждой итерации блока цикла.
- Инструкции `break` и `continue` могут использоваться **только внутри цикла**:
    - `break` — немедленно завершает текущий цикл.
    - `continue` — пропускает оставшуюся часть текущей итерации и переходит к проверке условия/шагу.

## 7. Типы данных

# Спецификация семантики типов данных (обновлённая)

Поддерживаются 5 типов данных:

* `int` — 64-битное целое число со знаком
* `float` - числа с плавающей точкой в десятичной системе счисления(как decimal)
* `string` — строковый тип, ASCII
* `bool` — булев тип (`true` / `false`)
* `void` — специальный тип для функций, не возвращающих значение

# Ключевые свойства языка

* Тип **обязательно указывается** при объявлении каждой переменной.
* **Тип переменной нельзя изменить** после объявления.
* **Неявные преобразования запрещены полностью.**
* **Явные преобразования запрещены**, кроме строго определённых функций:

    * `toString(x: int | bool): string`
    * `parseInt(s: string): int`
    * `toBool(x: int): bool`
    * `toFloat(x: int): float`

# Преобразование int → bool

Разрешено **только явно**, через функцию `toBool(x: int): bool`.

Правила:

* `toBool(0)` → `false`
* `toBool(x > 0)` → `true`
* `toBool(x < 0)` → `true`

# Тип void

Тип `void` существует только как возвращаемый тип функций.

* Функция **с оператором `return;`** имеет тип `void`.
* Если функция содержит хотя бы один `return <expression>`, она должна явно указывать возвращаемый тип, соответствующий выражению.
* Смешивать `return;` и `return value` запрещено.

## 8. Объявление и присваивание переменных
## Объявление переменной

* `var` — изменяемая переменная
* `const` — неизменяемая константа
* Тип **обязателен** и фиксируется навсегда
* Повторное объявление переменной с тем же именем в одной области видимости запрещено

## Присваивание

* Переменная должна быть **уже объявлена**
* Присваиваемое значение **должно соответствовать типу переменной**
* Присваивание переменной через выражение типа `void` запрещено
* Для `const` (констант) любое присваивание после инициализации запрещено

## 9. Операторы

## Арифметические (для int и float)

* `+` — сложение, также конкатенация для string
* `-` — вычитание
* `*` — умножение
* `/` — для `int` — целочисленное деление
  `/` — для `float` — деление с плавающей точкой

## Логические (для bool)

* `and` — логическое И, short-circuit evaluation
* `or` — логическое ИЛИ, short-circuit evaluation
* `!` — логическое отрицание

## Сравнения

* `==`, `!=` — для всех типов
* `<`, `<=`, `>`, `>=` —  для всех типов
* Для `string` — лексикографическое сравнение по ASCII

## Операторы для string

* `+` — конкатенация
* `len(s)` — вычисление длины, встроенная функция
* `substring(string, start, count)` — вычисление подстроки, встроенная функция

## 10. Прочие правила
- В левой части присваивания (`=`) может стоять только **идентификатор переменной или константы**.
- Обращение к **неинициализированной переменной** запрещено.
- Для `const` любое последующее присваивание после инициализации запрещено.
- Операторы `/`, `*`, `+`, `-` соблюдают строгую типизацию:
    - Если оба операнда `int`, результат `int` (целочисленное деление).
    - Если оба оператора `float`, результат `float` (деление с плавающей точкой).
- Логические операторы `and`, `or`, `!` применяются только к типу `bool`.
- Конкатенация строк (`+`) возможна только для `string`.
- Вызов встроенных функций должен соответствовать типам аргументов:
    - `toString(x: int | bool)` → `string`
    - `parseInt(s: string)` → `int`
    - `toBool(x: int)` → `bool`
    - `toFloat(x: int)` → `float`
- Главная функция main.
  - Функция main является точкой входа в программу — с неё начинается выполнение.
  - В программе должна быть ровно одна функция с именем main.
  - Функция main не принимает параметров и всегда возвращает тип int.
  - Возвращаемое значение main интерпретируется как код завершения программы:
    - 0 — программа завершилась успешно.
    - 1 (или любое ненулевое значение) — программа завершилась с ошибкой.
